# gdrive-mp4-extractor

A lightweight app for converting Google Drive share links into playback-friendly URLs.

It includes:
- a static web UI (`index.html`),
- optional serverless APIs (`/api/resolve`, `/api/mp4/:fileId`) for hosted/proxied playback,
- Vercel rewrite config for clean `/mp4/:fileId.mp4` URLs.

---

## Project structure

- `index.html` — client UI (paste link, generate URLs, copy/open).
- `lib/drive.js` — shared Drive parsing + stream resolution helpers.
- `api/resolve.js` — resolves/probes a Drive file and returns metadata + hosted URL.
- `api/mp4/[fileId].js` — streaming proxy endpoint.
- `vercel.json` — rewrite from `/mp4/:fileId.mp4` → `/api/mp4/:fileId`.

---

## Quick start (local)

```bash
git clone <your-repo-url>
cd gdrive-mp4-extractor
npm install
npm start
```

Then open:
- `http://localhost:3000`

This runs both the static UI and local API routes (`/api/resolve`, `/api/mp4/:fileId`, `/mp4/:fileId.mp4`).

---

## Deploy on Vercel

1. Push the repo to GitHub.
2. Import the project into Vercel.
3. Deploy.
4. Open your deployed URL and paste a Google Drive URL.

`vercel.json` already maps:
- `/mp4/:fileId.mp4` → `/api/mp4/:fileId`

---

## Optional environment variables

In Vercel (Project Settings → Environment Variables), you can provide:

- `GOOGLE_DRIVE_API_KEY`
- `GOOGLE_API_KEY`

Behavior:
- If an API key is present, the backend attempts Drive API media access first.
- If that fails, it falls back to public-link resolution logic.

---

## API reference

### `GET /api/resolve?input=<drive_url_or_file_id>`

Returns JSON:
- `fileId`
- `mp4Url`
- `validated` (boolean)
- `warning` (string)
- `contentType` (string|null)

### `GET /api/mp4/:fileId`

Streams the resolved Drive file through the serverless proxy.

Typical headers:
- `Content-Type`
- `Content-Length` (if upstream provides it)
- `Content-Disposition: inline`
- `Cache-Control`

## URL formats generated by UI

- VLC desktop: direct Drive download URL
- `vlc://` deep link
- `infuse://x-callback-url/play?...`
- `nplayer-https://...`

---

## Troubleshooting

- Ensure the Drive file is shared as **Anyone with the link**.
- Ensure downloads are allowed by the file owner.
- If a player fails, test the generated URL directly in a browser first.
- Some players handle Drive redirects/cookies differently.
